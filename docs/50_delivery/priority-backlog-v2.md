## 1. Objetivo

Traducir roadmap por fases a backlog accionable priorizado para equipo de 2-3 personas.

## 2. Backlog P0 (bloqueante)

1. Publicar matriz SoT por dominio en governance.
2. Cerrar fronteras de responsabilidad entre docs tecnicos clave.
3. Congelar alcance MVP (incluye/no incluye) en doc unico.
4. Definir contratos API minimos para loop principal.
5. Definir data model MVP (entidades y relaciones minimas).
6. Definir evento de analitica core para D1 y sesiones/dia.
7. Definir criterios de PASS para vertical slice.

## 3. Backlog P1 (critico de calidad)

1. Art bible MVP con paleta, tipografia y motion.
2. Reglas de presupuesto visual adaptativo por dispositivo.
3. Checklist de QA visual para PixiJS + UI HTML.
4. Tutorial contextual 3 pasos: criterios de exito.
5. Normalizacion de estados documentales en docs activos.

## 4. Backlog P2 (post-MVP)

1. Diseno de Fase 2: competitivo asincrono.
2. Diseno de implosion y meta-progresion.
3. Estrategia de monetizacion posterior a validacion.
4. Renaming completo a estandar ASCII (si se aprueba OQ-005).

## 5. Regla de priorizacion diaria

- Si hay A0 abierta, solo se trabaja en cerrar A0.
- Si no hay A0, ejecutar maximo 1 item P0 por persona.
- P1 solo avanza cuando no compromete P0 activo.

## 6. Formato de estado por item

Estados permitidos:

- Open
- In Progress
- Blocked
- Done

Cada item debe tener owner, evidencia de cierre y referencia documental.
