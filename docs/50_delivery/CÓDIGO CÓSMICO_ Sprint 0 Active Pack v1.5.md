## 1. Propósito

Convertir Sprint 0 en un plan operativo activo con objetivos de 2 semanas, tareas P0/P1, responsables por bloque y criterios diarios de seguimiento.

---

## 2. Ventana de Sprint 0

| Parámetro | Valor |
|---|---|
| Duración | 10 días hábiles |
| Objetivo central | Cerrar M0 y dejar M1 listo |
| Cadencia | Daily técnico + review semanal |
| Entorno | `dev` y `staging` |

---

## 3. Objetivos no negociables

1. Pipeline CI mínimo estable (lint + test smoke).
2. Esqueleto de módulos backend/frontend listo.
3. Backlog M1 en estado `ready` para al menos 80% de historias P0.
4. Riesgos críticos R-01/R-04/R-06 con mitigación activa documentada.

---

## 4. Scope de Sprint 0 (In)

### In scope

- estructura de repositorio y convenciones,
- automatización base de calidad,
- setup local reproducible,
- tableros de seguimiento y definición de handoff.

### Out of scope

- implementación completa de motor de simulación,
- integración full de UI,
- competitividad y economía runtime.

---

## 5. KPIs de Sprint 0

| KPI | Objetivo |
|---|---|
| Tareas P0 completadas | >= 90% |
| Fallos de pipeline en main/develop | 0 |
| Tiempo medio de desbloqueo de bloqueos | < 24h |
| Historias M1 en `ready` | >= 80% |

---

## 6. Criterio de éxito

Sprint 0 se considera exitoso cuando:

- [ ] M0 completado según milestones v1.3
- [ ] checklist de kickoff cerrada al 100%
- [ ] backlog M1 priorizado y listo para ejecución
- [ ] riesgo residual de arranque en nivel aceptable

---

## 7. Criterio de fracaso

Se considera fallido si ocurre cualquiera de:

- >2 tareas P0 bloqueadas sin owner activo,
- pipeline inestable por más de 2 días,
- desalineación entre backlog y milestones sin corrección.

---

## 8. Plan de contingencia rápido

1. Congelar nuevas tareas P2/P3.
2. Repriorizar bloqueos técnicos críticos.
3. Ejecutar mini-sprint de estabilización de 2 días.
4. Replanificar M1 con alcance recortado si aplica.

---

## 9. Entregables de cierre

- reporte de cierre Sprint 0,
- tablero actualizado y snapshot de estado,
- release notes documental v1.5,
- confirmación de paso a M1.

---

## 10. Declaración Final

Sprint 0 Active Pack v1.5 es el contrato operativo para iniciar construcción con control de calidad y velocidad. Si no se cumple este contrato, no se escala a implementación de M1.
