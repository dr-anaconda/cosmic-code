## 1. Propósito

Este documento establece la visión de producto, define el público objetivo, establece las métricas de éxito y delimita el alcance del MVP para **Código Cósmico – Evolución Profunda**.

- ¿Por qué existe este producto?

## 2. Visión del Producto

### 2.1 Declaración de Visión

**Código Cósmico** es un juego híbrido que combina simulador de dios, estrategia asíncrona e idle game, donde el jugador asume el rol de una conciencia primordial capaz de sembrar vida en múltiples planetas y guiar su evolución desde bacterias hasta civilizaciones espaciales.

El producto se diferencia en el mercado por:

1. **Profundidad evolutiva real**: No es un idle game pasivo, sino una simulación donde las decisiones del jugador tienen consecuencias sistémicas visibles en la evolución de especies.

2. **Ciclo multiversal**: El reinicio (implosión) no es un castigo, sino una mecánica de progresión que otorga bonificadores permanentes, creando un loop de "partida nueva + ventajas acumuladas" único en el género.

3. **Narrativa emergente**: No hay guión escrito. Las historias surgen de la interacción de sistemas complejos: una bacteria resistente a la radiación, una civilización que colapsa por contaminación, un ecosistema que se recupera de una extinción masiva.

4. **Multijugador asíncrono social**: Las guerras de ecosistemas, los torneos semanales y las interacciones entre civilizaciones crean un metajuego sin necesidad de coordinación en tiempo real.

### 2.2 Posicionamiento en el Mercado

El producto ocupa un espacio única en la intersección de:

| Género base | Elemento diferenciador |
|-------------|----------------------|
| Simulador de dios | Evolución procedural con redes neuronales simplificadas |
| Idle game | Consecuencias sistémicas, no progresión lineal |
| Estrategia asíncrona | Torneos semanales con rankings |
| TCG procedural | Creación de especies mediante combinación genética |

### 2.3 Propuesta de Valor Única

> "Siembra vida. Guía su destino. Trasciende el universo."

El jugador no controla directamente a las criaturas, sino que crea las condiciones para que la vida florezca, compita y evolucione. La satisfacción viene de presenciar historias emergentes únicas que nunca se repetirán igual.

## 3. Público Objetivo

### 3.1 Perfil Principal: "El Explorador Sistémico"

| Característica | Descripción |
|---------------|-------------|
| **Edad** | 18-45 años |
| **Plataforma principal** | Móvil (iOS/Android) |
| **Plataforma secundaria** | Navegador web (usuarios más hardcore) |
| **Experiencia gamer** | Moderada; conoce idle games o strategy games |
| **Motivación** | Exploración, creación, ver cómo "cosas pasan" |
| **Tiempo disponible** | Sesiones de 5-15 minutos varias veces al día |
| **Dispuesto a esperar** | Sí, si hay progreso visible |

**Comportamiento típico:**

- Abre la app por la mañana y antes de dormir
- Ve las barras de progreso avanzar
- Toma decisiones rápidas (terraformar, crear especie)
- Cierra la app y espera
- Vuelve a las pocas horas para ver resultados

### 3.2 Perfil Secundario: "El Estratega Competitivo"

| Característica | Descripción |
|---------------|-------------|
| **Edad** | 21-35 años |
| **Plataforma** | Web (más pantalla para análisis) |
| **Experiencia gamer** | Alta; ha jugado strategy games, TCGs |
| **Motivación** | Competir, optimizar, estar en top de rankings |
| **Tiempo disponible** | Sesiones más largas (30-60 min) |
| **Análisis** | Estudia stats, planifica build orders |

**Comportamiento típico:**

- Dedica tiempo a analizar sus ecosistemas
- Participa activamente en torneos semanales
- Comparte estrategias en comunidad
- Invierte en optimización de Genética + Guerra

### 3.3 Perfil Terciario: "El Narrativista"

| Característica | Descripción |
|---------------|-------------|
| **Edad** | 25-50 años |
| **Plataforma** | Móvil |
| **Motivación** | Ver qué historia emerge esta partida |
| **Tiempo disponible** | Variable, más casual |
| **Enfoque** | Disfruta el proceso, no solo el resultado |

### 3.4 Audiencia a Excluir

**No es para:**

- Jugadores que buscan acción en tiempo real
- Jugadores que requieren feedback inmediato constante
- Jugadores que buscan simulación científica realista
- Jugadores que se frustran con RNG (aunque las decisiones importan más que la suerte)

## 7. Alcance del MVP

### 7.1 Qué Incluye

**Core Gameplay:**

- Un planeta con bacteria inicial
- Sistema de evolución niveles 1-4 (bacterias → organismos pluricelulares simples)
- Editor genético con 8-10 rasgos básicos
- Mutaciones aleatorias visibles
- Árbol genealógico básico

**Progresión:**

- Puntos de conocimiento por hitos
- Árbol de habilidades (solo rama Genética nivel 1)
- Bonificadores de implosión (sin implosión real aún)

**Idle:**

- Barras de progreso en tiempo real
- Cuenta atrás para hitos
- Notificaciones push (opt-in)

**UI/UX:**

- Vista planeta con zoom
- Dashboard de stats
- Editor genético (modal)
- Bolsa de eventos/notificaciones

### 7.2 Qué NO Incluye (a propósito)

| Exclusión | Razón |
|-----------|-------|
| Multiplaneta | Complejidad excesiva para MVP |
| Multijugador | Requiere backend robusto |
| Torneos | Requiere masa de usuarios |
| Dimensiones 4-5 | Contenido late-game |
| Tienda de tokens | Monetización post-MVP |
| Animaciones complejas | Performance first |
| Mapas estelares | Dimensión 1 fuera de alcance |

**El MVP debe ser jugable, rejugable y demostrativo del concepto, no completo.**

## 9. Riesgos de Producto

| Riesgo | Probabilidad | Impacto | Mitigación |
|--------|-------------|---------|------------|
| Complejidad sistémica abruma usuarios | Alta | Alto | Tutorial progresivo, UI contextual |
| Retención baja en idle games | Media | Alto | Eventos frecuentes, progresión visible |
| Equilibrio de ecosistemas difícil | Media | Medio | Simulaciones, A/B testing |
| Competencia en género | Baja | Medio | Diferenciación clara |
| Costes server elevados | Media | Medio | Arquitectura server-light |
